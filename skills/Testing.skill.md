# Testing Skill

## Overview

Testing is a first-class artifact in AEOS. This skill documents the principles and patterns needed to make tests reliable, maintainable, and meaningful.

## Core concepts

- Unit tests for isolated behavior.
- Integration tests for component interaction.
- Regression tests for bug fixes.
- Validation of documentation and examples.

## Best practices

- Write tests before or alongside implementation.
- Keep tests small and focused.
- Make expected behavior explicit in test names.
- Use test artifacts as part of the project documentation.
