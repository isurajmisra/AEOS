# Release Checklist

- [ ] Confirm the release version and codename.
- [ ] Update `VERSION` and release notes if needed.
- [ ] Verify that documentation is current.
- [ ] Ensure all relevant ADRs are recorded.
- [ ] Confirm tests pass for the release scope.
- [ ] Validate that the repository structure follows AEOS conventions.
- [ ] Create or update release artifacts and examples.
- [ ] Publish the release notes and announce the release.
