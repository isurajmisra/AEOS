# New Feature Workflow

## Purpose

Define a consistent workflow for delivering new features in AEOS.

## Steps

1. Understand the goal and scope.
2. Review relevant context: `README.md`, `INDEX.md`, `MANIFEST.md`, architecture, standards, skills.
3. Draft a design or architecture approach.
4. Create an ADR if the decision is significant.
5. Implement the feature in small, reviewable increments.
6. Add or update tests as first-class artifacts.
7. Update documentation and examples.
8. Submit a pull request with a clear summary and checklist.

## Review criteria

- The change is small and understandable.
- The implementation follows relevant standards.
- Documentation and tests are present.
- The expected behavior is easy to verify.
