# Release Workflow

## Purpose

Define AEOS release activities and gate criteria.

## Steps

1. Confirm the release version and codename.
2. Review the `MANIFEST.md` for scope and goals.
3. Verify documentation is current and complete.
4. Ensure all required ADRs are recorded.
5. Run release validation and tests.
6. Complete the release checklist.
7. Publish the release artifacts.
8. Announce the release and update status documents.

## Review criteria

- The release scope is clearly defined.
- All work is traceable through ADRs, docs, or examples.
- The repository structure and naming conventions are respected.
