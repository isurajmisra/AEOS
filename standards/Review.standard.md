# Review Standard

## Purpose

Reviews are the quality gate for AEOS. They ensure that changes are understandable, correct, and aligned with project standards.

## Rules

- Keep pull requests small and focused.
- Review for correctness, readability, and maintainability.
- Confirm tests and documentation accompany meaningful changes.
- Use the implementation review checklist for consistency.
- Provide constructive feedback and approve when the change is ready.

## Review process

- Check the README, INDEX, and MANIFEST for relevant context.
- Validate the change against applicable standards and workflows.
- Ensure decision-making is recorded when needed.
- Avoid scope creep by separating unrelated work into new changes.
