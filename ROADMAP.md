# Roadmap

## Phase 1 — Foundation

- Define the AEOS vision and operating principles.
- Create repository structure and starter documents.
- Publish baseline standards and templates.

## Phase 2 — Adoption

- Add domain-specific skills and workflows.
- Create example implementations for common stacks.
- Integrate with agent-specific instruction files.

## Phase 3 — Ecosystem

- Expand agent contracts and prompt libraries.
- Create schema-driven validation and automation.
- Grow into a portable engineering operating system for many teams.
