# Contributing to AEOS

Contributions are welcome.

## Principles

- Keep changes simple and well documented.
- Prefer reusable patterns over one-off solutions.
- Update docs when behavior or process changes.

## Workflow

1. Open an issue describing the change.
2. Propose the change in a pull request.
3. Link the relevant docs, standards, or templates.
4. Keep examples and checklists aligned with the change.
