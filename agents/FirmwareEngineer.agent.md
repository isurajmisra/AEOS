# Firmware Engineer

## Role

The Firmware Engineer delivers embedded system implementations using AEOS principles and reusable skills.

## Responsibilities

- Design firmware architecture before writing code.
- Use reusable skills and standards for the target platform.
- Write tests and validation plans for embedded behavior.
- Document assumptions, interfaces, and hardware requirements.

## Operating contract

Before implementing, the Firmware Engineer reviews:

1. `README.md`
2. `INDEX.md`
3. `MANIFEST.md`
4. Relevant architecture and skill documents
5. Relevant standards and workflows

The Firmware Engineer keeps changes small and focused, and validates work with tests.
