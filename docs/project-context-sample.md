# Sample Project Context

## Project overview

- Name: AEOS Genesis
- Description: A foundational AI Engineering Operating System for defining standards, workflows, agents, and reusable knowledge.
- Scope: Establish a minimal, discoverable repository structure and delivery model.
- Target audience: engineering teams adopting AI-native workflows.

## Goals

- Primary goal: Define the AEOS core philosophy and initial artifact set.
- Secondary goals: Establish naming conventions, execution order, and review practices.
- Success criteria: A coherent set of documents and templates that can be used by humans and AI agents.

## Architecture summary

- Key systems: Vision, Principles, Standards, Agents, Skills, Workflows, Templates, Checklists, Examples.
- Important constraints: Keep the repository human-readable and AI-discoverable.
- Relevant standards: Documentation.standard.md, Naming.standard.md, Review.standard.md.

## Delivery context

- Current phase: Initial release and foundation build.
- Key stakeholders: Chief AI Engineer, Product Manager, Firmware Engineer, Reviewer.
- Important dependencies: Clear naming conventions and a central execution model.

## Quality and release

- Testing expectations: All meaningful changes should include validation or review evidence.
- Release cadence: Incremental, focused releases beginning with Genesis.
- Documentation requirements: Every decision and artifact should be documented or linked.
