# AEOS Operating Model

AEOS is designed as a layered engineering system with a stable foundation and modular runtime.

## Layers

- Foundation: vision, mission, principles, philosophy
- Standards: code, review, documentation, architecture
- Runtime: agents, workflows, prompts, skills
- Delivery: templates, checklists, examples, schemas

## Domain organization

Each domain is a deliberate container for knowledge:

- Vision: project intent and long-term goals.
- Principles: the values that guide engineering decisions.
- Standards: shared expectations and rules.
- Agents: role definitions and operating contracts.
- Skills: reusable domain knowledge.
- Workflows: repeatable processes for delivery.
- Templates: consistent artifacts for documentation and coordination.
- Checklists: quality gates and release readiness.
- Examples: reference patterns and usage guides.

## Discovery model

To support both humans and AI agents, AEOS uses predictable file naming and a central loading order. This reduces ambiguity and makes context easy to find.
