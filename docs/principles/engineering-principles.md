# Engineering Principles

AEOS defines its operating philosophy through the following principles:

1. AI augments engineers. It does not replace them.
2. Design before code. Architecture is decided before implementation begins.
3. Documentation is part of the product. Good documentation is a deliverable, not a burden.
4. Every important decision has a reason. Significant choices are recorded as ADRs.
5. One source of truth. Knowledge should not be duplicated across scattered files or habits.
6. Standards over preferences. Consistency matters more than individual style.
7. Small, reviewable changes. Pull requests should be easy to inspect and reason about.
8. Tests are first-class artifacts. They validate behavior and protect future change.
9. Everything should be reusable. Agents, prompts, templates, standards, and skills are all assets to be shared.
10. Human-first. AI helps, but humans make the final decisions.

## Practical interpretation

These principles guide architecture, documentation, delivery, and review. They are intended to be applied consistently across projects and teams.
